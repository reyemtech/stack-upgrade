**plan.md** — Header: "CRA to Vite Migration Plan", Date: ${UPGRADE_DATE}, Repo: ${REPO_URL}. Then 8 sequential phases:

Phase 1: Viject Scaffold (CRA-01)
- Run `npx viject` to scaffold Vite configuration
- Commit viject output immediately — clean git history showing what the tool did
- If viject fails: manually install vite + @vitejs/plugin-react, create vite.config.ts, move index.html from public/ to root with `<script type="module" src="/src/index.jsx">`, update package.json scripts (start→vite, build→vite build, test→keep as-is for now)
- Do NOT fix REACT_APP_ vars yet — that's Phase 2
- Verify: project structure looks correct, `npm install` / `pnpm install` succeeds

Phase 2: Env Var Rewrite (CRA-02)
- Find ALL `REACT_APP_` references: `grep -r "REACT_APP_" src/ .env .env.local .env.development .env.production`
- Rewrite in source files: `process.env.REACT_APP_X` → `import.meta.env.VITE_X`
- Rewrite in .env files: `REACT_APP_X=value` → `VITE_X=value`
- Also check for `process.env.NODE_ENV` — Vite uses `import.meta.env.MODE` (but `process.env.NODE_ENV` still works in Vite via define)
- Verify: `grep -r "REACT_APP_" src/ .env*` returns no results

Phase 3: Index HTML Relocation (CRA-03)
- viject usually handles this — verify index.html is in project root (not public/)
- Ensure `<script type="module" src="/src/index.jsx">` (or .tsx) tag is present
- Remove `%PUBLIC_URL%` references in index.html — Vite doesn't use this
- Verify: index.html in root with module script tag, no %PUBLIC_URL%

Phase 4: SVG Import Configuration (CRA-04)
- Check if project imports SVGs as React components (`import { ReactComponent as X }` or `import X from './x.svg'` used as JSX)
- If yes: `npm install -D vite-plugin-svgr`, add `svgr()` to vite.config plugins
- Update SVG imports if needed: CRA uses `import { ReactComponent as Logo }` → Vite with svgr can keep this pattern if configured correctly
- Verify: `vite build` passes without SVG import errors

Phase 5: Proxy Config Translation (CRA-05)
- Check if `src/setupProxy.js` exists
- If it uses simple `createProxyMiddleware(path, { target })` calls: translate to `server.proxy` in vite.config
- If it uses Express-specific methods (`app.get`, `app.post`, custom middleware): log complexity in run-log.md, attempt translation, mark failed after 3 attempts if too complex
- Remove setupProxy.js after successful translation
- Verify: dev server proxy works (or phase marked failed with documentation)

Phase 6: Build Verification (CRA-06)
- Run `.upgrade/scripts/verify-full.sh` — full build + lint + tests + REACT_APP_ grep
- Fix any remaining build errors
- Ensure react-scripts is fully removed from package.json
- Verify: `.upgrade/scripts/verify-full.sh` passes clean, react-scripts not in package.json

Phase 7: CI/CD Env Var Flagging (CRA-07)
- Grep `.github/workflows/`, `Dockerfile`, `docker-compose.yml`, `.gitlab-ci.yml`, `Jenkinsfile`, `Makefile`, and other CI configs for `REACT_APP_` references
- For each found: add a comment/entry in `.upgrade/run-log.md` and `.upgrade/changelog.md` noting the file and line that needs manual update
- Do NOT modify CI/CD files (they may have deployment secrets context)
- Verify: all CI/CD REACT_APP_ references are documented

Phase 8: Jest to Vitest Migration (CRA-08)
- Install vitest: `npm install -D vitest @testing-library/jest-dom`
- Add test config to vite.config: `test: { globals: true, environment: 'jsdom', setupFiles: './src/setupTests.ts' }`
- Replace `jest.fn()` → `vi.fn()`, `jest.mock()` → `vi.mock()`, `jest.spyOn()` → `vi.spyOn()` across test files
- Update package.json test script: `"test": "vitest"` (or `"test": "vitest run"` for CI)
- Uninstall jest and @types/jest
- Check for jest.config.js — if complex transforms exist, log in run-log.md
- Verify: `npm test` / vitest passes, jest is not in package.json

End with Constraints section.
