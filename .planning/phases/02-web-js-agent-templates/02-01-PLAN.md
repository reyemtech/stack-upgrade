4. **Constraints:**
   - Upgrade to target version — the goal is staying on supported major versions
   - Never change application behaviour
   - Never delete tests — fix them for new APIs
   - `UnsafeUnwrapped` markers and `@next/codemod` comments are BUILD BLOCKERS, not warnings. The agent must resolve all of them before marking a phase complete. verify-fast.sh exits non-zero if these exist.

   - Skip unused packages — remove instead of upgrading
   - Never force-install incompatible packages

5. **Reference Material:**
   - `.upgrade/nextjs-upgrade-guide.html` — fetched at runtime, version-specific
   - `.upgrade/recon-report.md` — pre-analyzed repo overview

   - `.upgrade/scripts/verify-fast.sh` — lint + tests + tsc + grep checks for UnsafeUnwrapped/codemod markers (run frequently)
   - `.upgrade/scripts/verify-full.sh` — above + build + npm audit (run before phase completion)

7. **Useful Commands:**
   ```bash
   npx @next/codemod@canary upgrade latest
   npx @next/codemod@canary upgrade 15
   npx @next/codemod@canary upgrade 16
   next build
   next start  # for middleware smoke testing
   ```

8. **Middleware Verification Note:** Next.js middleware/proxy runs at Edge runtime and cannot be tested with Jest/Vitest. **plan.md** — Follow Laravel plan.md structure. Header with Target/Date/Repo, then sequential phases:

Phase 1: Codemod (NEXT-01)
- Run `npx @next/codemod@canary upgrade ${TARGET_NEXTJS}`
- Commit codemod output immediately — clean git history showing what the tool did
- Do NOT manually fix anything the codemod leaves behind (that's Phase 2)
- Verify: `.upgrade/scripts/verify-fast.sh` (may fail on UnsafeUnwrapped — expected at this stage)

Phase 2: Async Request API Migration (NEXT-02)
- Resolve ALL `UnsafeUnwrapped` markers left by codemod
- Fix all `@next/codemod` comment markers
- Ensure params, searchParams, cookies(), headers() are properly awaited
- Verify: `.upgrade/scripts/verify-fast.sh` — must pass (no UnsafeUnwrapped/codemod markers)

Phase 3: Middleware / Proxy Migration (NEXT-03)
- If upgrading to Next.js 16+: handle middleware.ts → proxy.ts rename (codemod may have done this)
- Check if project uses Edge runtime in middleware — if so, keep middleware.ts (Edge not supported in proxy)
- Verify: `next build` passes, middleware/proxy file is correct

Phase 4: Turbopack Configuration (NEXT-04)
- If next.config has `experimental.turbopack`, move to top-level `turbopack` key
- If next.config has `webpack:` config: either migrate to `turbopack:` equivalent OR add `--webpack` to build script
- Verify: `.upgrade/scripts/verify-fast.sh`

Phase 5: Cache Semantics Audit (NEXT-06)
- If upgrading from v14 or earlier: scan for fetch() calls relying on implicit caching
- Add explicit `cache: 'force-cache'` or `next: { revalidate: N }` where implicit caching was relied upon
- Check for route segment configs that need updating
- Verify: `.upgrade/scripts/verify-fast.sh`

Phase 6: Build Verify + Grep Checks (NEXT-05, NEXT-07)
- Run `.upgrade/scripts/verify-full.sh` — full build + lint + tests + tsc + grep checks
- Resolve any remaining codemod markers, UnsafeUnwrapped, deprecated config keys
- Verify: `.upgrade/scripts/verify-full.sh` passes clean

Phase 7: Dependency Cleanup
- Run `npm outdated` / equivalent for pkg manager
- Update remaining outdated dependencies (React, React DOM, types)
- Remove unused packages
- Final `.upgrade/scripts/verify-full.sh`

End with a Constraints section matching CLAUDE.md.
