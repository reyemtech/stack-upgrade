**verify-fast.sh** (`stacks/node/scripts/verify-fast.sh`):
Per CONTEXT.md locked decisions and research Pattern 5. Must detect tools automatically and skip gracefully when absent.

```bash
#!/bin/bash
set -e
echo "=== verify-fast ==="
cd /workspace
```

**Section 1: Lint** (auto-detect biome or eslint)
- Check `biome.json` or `biome.jsonc` -> run `npx biome check .`
- Else check `.eslintrc*` or `eslint.config.*` or eslint in package.json devDeps -> run `npx eslint . --max-warnings=0`
- Else: log "No lint tool detected -- skipping"

**Section 2: Tests** (auto-detect vitest, jest, or npm test)
- Check vitest in devDeps/deps -> `npx vitest run`
- Else check jest in devDeps/deps -> `npx jest --passWithNoTests`
- Else check `.scripts.test` exists and is not "no test" -> `npm test -- --watchAll=false 2>&1 || true`
- Else: log "No test runner detected -- skipping" (per CONTEXT: skip with warning, don't fail)

**Section 3: TypeScript** (if tsconfig.json exists)
- `npx tsc --noEmit`

**Section 4: Post-build grep checks** (INFRA-08 — silent-failure markers)
These run even if no build output exists yet (just skip with log). Use detect-stack.sh to know which checks apply.

Get STACK_TYPE from env or call detect-stack.sh:
```bash
STACK_TYPE="${STACK_TYPE:-$(/skill/scripts/detect-stack.sh 2>/dev/null || echo "unknown")}"
```

If `dist/` or `.next/` or `build/` directory exists:
- CRA/vite-react: `grep -r "REACT_APP_" dist/ build/ 2>/dev/null | grep -v node_modules` -> if found, ERROR and exit 1 ("env vars not migrated to VITE_")
- nextjs: `grep -r "UnsafeUnwrapped\|@next/codemod" src/ app/ pages/ 2>/dev/null | grep -v node_modules` -> if found, ERROR and exit 1 ("codemod markers not resolved")

End with: `echo "=== verify-fast PASSED ==="`

**verify-full.sh** (`stacks/node/scripts/verify-full.sh`):
Runs verify-fast first, then build + audit.

```bash
#!/bin/bash
set -e
echo "=== verify-full ==="
cd /workspace

# Run verify-fast first
/skill/scripts/verify-fast.sh

# Build
PKG_MANAGER="${PKG_MANAGER:-npm}"
echo "[build] Running build..."
case "$PKG_MANAGER" in
  pnpm) pnpm run build ;;
  yarn) yarn build ;;
  npm)  npm run build ;;
esac

# npm audit (high/critical only per CONTEXT decision)
echo "[audit] Running security audit..."
case "$PKG_MANAGER" in
  pnpm) pnpm audit --prod 2>/dev/null || true ;;
  yarn) yarn audit --level high 2>/dev/null || true ;;
  npm)  npm audit --audit-level=high 2>/dev/null || true ;;
esac

echo "=== verify-full PASSED ==="
```

Note: audit uses `|| true` because audit exit codes signal "vulnerabilities found" which is informational, not a build failure. The agent logs the output but doesn't block on it.

Both scripts must have `#!/bin/bash` shebang and be chmod +x.

**recon.sh** (`stacks/node/scripts/recon.sh`):
Produces `.upgrade/recon-report.md` with deep analysis per CONTEXT.md locked decisions.

```bash
#!/bin/bash
set -e
cd /workspace
REPORT=".upgrade/recon-report.md"
mkdir -p .upgrade
```

**Shared base sections** (always run):

1. **Header**: `# Recon Report` + date + detected stack type
2. **Stack Detection**: Call `/skill/scripts/detect-stack.sh`, display result
3. **Package Manager**: Detect from lockfile (pnpm-lock.yaml > yarn.lock > package-lock.json > npm default). Display which PM and lockfile.
4. **Framework Versions**: Extract key framework deps from package.json using `jq`:
   - react, react-dom, next, vite, @vitejs/plugin-react, react-scripts, typescript
   - Display version for each that exists
5. **Dependencies Overview**: Count of deps + devDeps. List all deps with versions using `jq`.
6. **Test Runner**: Detect vitest/jest/npm-test. Count test files (`find . -name "*.test.*" -o -name "*.spec.*" | grep -v node_modules | wc -l`).
7. **Build Tool**: Detect from package.json scripts (next build, vite build, react-scripts build, custom).
8. **TypeScript**: Check tsconfig.json, report version.
9. **Lockfile Type**: Which lockfile exists.
10. **Outdated Packages**: `$PKG_MANAGER outdated --json 2>/dev/null || true` — parse with jq if available, fallback to raw text. NOTE: `npm outdated` exits 1 when packages are outdated; always use `|| true`.
11. **Environment Variables**: Scan source for `process.env.` and `import.meta.env.` usage patterns. Report count and unique var names.
12. **Custom Plugins**: Check vite.config.* and webpack.config.* for plugin usage (list plugin names if parseable).

**Stack-specific sections** (gated by STACK_TYPE):

- **nextjs**: Check next.config.* format (mjs/js/ts), middleware.ts presence, app/ vs pages/ router, image optimization config
- **cra**: Check for setupProxy.js, custom-react-scripts, ejected status (check for config/ directory with webpack configs)
- **vite-react**: Check vite.config format, plugin list, CSS preprocessor (sass/less/stylus in deps)

Write all output to `$REPORT` using `echo "..." >> "$REPORT"` with markdown formatting.

Use `PKG_MANAGER` env var (exported by entrypoint) or detect inline as fallback.

Handle all commands defensively with `2>/dev/null || true` or `|| echo "..."` — recon must never fail the entrypoint.
